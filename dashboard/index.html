LG

Single line-to-ground

One phase conductor contacts earth through fault impedance. Most common distribution fault, accounting for roughly 70–80% of all events.

Rf range · 0 to 1.0 pu
LL

Line-to-line

Two phase conductors short directly without involving ground. Common in storm conditions when phase conductors swing together.

Rf range · 0 to 1.0 pu · phases B and C
LLG

Double line-to-ground

Two phase conductors simultaneously fault to ground. Common from fallen tree branches contacting multiple conductors.

Rf range · 0 to 1.0 pu · phases B and C
LLLG

Three-phase-to-ground

All three phase conductors faulted to ground simultaneously. Rare but the most severe condition — symmetrical fault with the highest fault current.

Rf range · 0 to 1.0 pu · balanced
HIF

High-impedance fault

Fault current limited by very high impedance — vegetation, surface contact, or downed lines on dry ground. Conventional overcurrent protection often misses these.

Rf range · 2.0 to 2000 pu

What this tells us

Fault type boundaries are sharp and tree-separable — ensemble methods reach near-perfect accuracy. Fault location is a continuous spatial problem where the neural network's representation learning produces the cleanest decision surface. SHAP attribution confirms the models rely on physically meaningful features: zero-sequence components for ground faults, negative-sequence for unbalanced faults, and voltage angles for high-impedance detection.

Method

How the pipeline works.

From IEEE 33-bus simulation through sequence-network fault analysis to a trained classifier — every step is reproducible and grounded in classical power systems theory.

Pipeline
01

IEEE 33-bus network

The Baran and Wu radial distribution feeder — 33 buses, 32 branches, 12.66 kV line-to-line, 3.715 MW total load. A widely cited benchmark in distribution research.

02

Sequence-network fault analysis

Faults injected through Y-bus matrix modification using symmetrical components theory. Positive, negative, and zero-sequence networks solved for each scenario based on the fault type.

03

Backward-forward sweep

Pre-fault voltages computed via radial distribution load flow with tolerance 1e-10. Post-fault voltages derived from sequence network equations and converted to phase values.

04

Balanced training set

Exactly 320 scenarios per class. Normal operation rows generated across 320 varied loading conditions (55–145% of base) to capture realistic operating diversity.

05

Model training and SHAP attribution

Three models trained with 80/20 stratified split. SHAP TreeExplainer applied to validate that models learned physically correct features rather than spurious correlations.

Feature engineering

V_bus Voltage magnitude

Per-unit RMS voltage at each bus. Captures the voltage sag that defines fault severity and electrical distance from the fault location.

Angle_bus Voltage angle

Phase angle relative to the reference bus. Carries critical information for high-impedance fault detection where magnitude alone is insufficient.

V0_bus Zero sequence

The component that exists only when a ground path is present. Spikes during LG, LLG, and HIF; theoretically zero for balanced operation and LL faults.

V1_bus Positive sequence

The fundamental balanced operating voltage. Reduces during every fault type, providing a baseline indication that a fault has occurred.

V2_bus Negative sequence

The unbalance indicator. Rises sharply during asymmetric faults (LG, LL, LLG); near zero for balanced faults (LLLG) and normal operation.

Scope and limitations

This system is trained on steady-state simulation data from the IEEE 33-bus benchmark network. Performance on real grid measurements may differ due to instrumentation noise, harmonic distortion, and operating conditions not represented in the simulation. The model is specific to this topology and will not transfer directly to other networks without retraining. Zero-sequence impedance was approximated as Z0 = 3·Z1 uniformly across all lines — a standard assumption when actual zero-sequence data is unavailable, but one that should be revisited for utility-grade deployment.
